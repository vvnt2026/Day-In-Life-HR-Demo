# Vercel Deployment Audit Report

## 1. Detected Tech Stack
- **Framework**: TanStack Start (React-based full-stack framework)
- **Routing**: @tanstack/react-router
- **Bundler**: Vite
- **Styling**: Tailwind CSS v4
- **UI Components**: Radix UI Primitives (shadcn/ui-like setup)
- **Icons**: Lucide React
- **Language**: TypeScript
- **State Management**: @tanstack/react-query

## 2. Issues Found
The project was generated by Lovable with a strong assumption that it will be deployed to **Cloudflare**. This is evident from the configuration files and will cause Vercel deployments to fail or return 404 errors.

**Specific Issues**:
1. **Cloudflare-Specific Vite Configuration**: The `vite.config.ts` imports `@lovable.dev/vite-tanstack-config`, which hardcodes the `@cloudflare/vite-plugin`. This forces the build output to target Cloudflare Workers rather than Vercel's Build Output API.
2. **Missing Vercel Output Structure**: Vercel expects server functions to be placed in `.vercel/output/functions/__server.func` (or similar, depending on the Vercel preset). The current config outputs for Cloudflare.
3. **Unnecessary Cloudflare Files**: `wrangler.jsonc` is present, which is entirely useless for Vercel.
4. **Static Assets Handling**: There is no explicit `public/` directory found in the root. If you add static assets (images, fonts), they need to be properly placed in `public/` or imported in Vite so Vercel can serve them statically.

## 3. Recommended Changes
1. **Remove Cloudflare Configuration**: 
   - Delete `wrangler.jsonc`.
   - Update `vite.config.ts` to either override the Nitro preset to `vercel` or remove the Lovable Cloudflare wrapper entirely and use standard TanStack Start Vite plugins.
2. **Configure Vercel Build Output**: Ensure the build process outputs to the `.vercel/output` directory natively or configure Vercel's output directory to point to standard `dist` if running as a static SPA. However, since `src/server.ts` exists, TanStack Start SSR is enabled, so the `vercel` Nitro preset is required.
3. **Add `vercel.json` (Optional but recommended)**: To ensure all routes correctly fall back to the TanStack Start server function if needed.

## 4. Reason for Each Change
- **Removing Cloudflare dependencies**: Prevents the build system from compiling the server entry for Cloudflare Workers, which Vercel's edge/serverless environments do not understand.
- **Using Vercel Preset / Build Output**: Allows Vercel to automatically detect the serverless functions and static assets, preventing the persistent `404 Not Found` error.
- **`vercel.json`**: Explicitly maps routes and configures cache headers, giving you control over Vercel's routing engine.

## 5. Potential Risks
- **Overriding Lovable's Config**: Since `@lovable.dev/vite-tanstack-config` is a black-box wrapper, bypassing it might remove some hidden aliases or plugins. However, it's necessary for Vercel deployment.
- **Serverless Function Limits**: TanStack Start SSR functions will run as Vercel Serverless Functions. Ensure that any backend logic (if added later) adheres to Vercel's 10-second execution limit on the Hobby tier.
- **Environment Variables**: Ensure any `VITE_*` env vars are migrated to Vercel's dashboard.
